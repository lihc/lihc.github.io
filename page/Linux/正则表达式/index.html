正则表达式

正则表达式 (Regular Expression) 描述了一种字符串匹配的模式（pattern），可以用来检查一个串是否含有某种子串、将匹配的子串替换或者从某个串中取出符合某个条件的子串等。

基本正则表达式

基本正则表达式元字符集合及其意义：

符号意义
*0 个或多个在 * 字符之前的普通字符
.匹配任意字符
^匹配行首，在 [] 中代表非
$匹配行尾
[]匹配字符集合
\转义符，屏蔽一个元字符的特殊意义
\<\>精确匹配符号
\{n\}匹配前面字符出现 n 次
\{n,\}匹配前面字符少出现 n 次
\{n,m\}匹配前面字符出现 n ~ m 次

1. “ * ” 符号

“ * ” 符号用于匹配前面一个普通字符的 0 次或多次重复。

hel*o
# 匹配如下字符
hello
helllo
helllllllo
...

2. “ . ” 符号

“ . ” 符号用于匹配任意一个字符。

_73.
# 匹配 _73 后面加任意一个字符

3. “ ^ ” 符号

“ ^ ” 符号用于匹配行首，匹配以指定字符开头的行。

^root
# 匹配以 root 开头的行

4. “ $ ” 符号

“ $ ” 符号用于匹配行尾，匹配以指定字符结尾的行。

bash$
# 匹配以 bash 结尾的行

5. “ [] ” 符号

“ [] ” 符号用于匹配字符集合，支持穷举列出字符集合所有元素，也支持 “ - ” 符号表示字符集合范围。

# 匹配任意一个数字
[0123456789]
[0-9]

# 匹配任意一个字母
[a-z]
[A-Z]

“ ^ ” 字符表示已指定字符开头，但是，当 “ ^ ” 符号放到 “ [] ” 中则表示取反。

[^A]
# 不匹配 A

6. “ \ ” 符号

“ \ ” 表示转义字符，用于屏蔽一个元字符的特殊意义。

# 匹配 "." 符号
\.

7. “ \<\> ” 符号

“ \<\> ” 符号表示精确匹配，该符号利用 “ \ ” 屏蔽 “ <> ” 符号。

\<the\>
# 精确匹配 the , 不匹配包含 the 字符的单词, 如 them、there 等。

8. “ \{\} ” 符号

JO\{3\}B
# 匹配重复字符 O 3 次，匹配值为: JOOOB
JO\{3,\}B
# 匹配重复字符 O 至少 3 次，匹配值为: JOOOB、JOOOOB、JOOOOOB...
JO\{3,6\}B
# 匹配重复字符 O 3~6 次，匹配值为: JOOOB、...JOOOOOOB
[a-z]\{5\}
# 精确匹配 5 个小写字母，如：hello、house等

扩展正则表达式

符号意义
？匹配 0 个或 1 个在其之前的普通字符
+匹配 1 个或多个在其之前的普通字符
()表示一个字符集合或用在 expr 中
|表示 “或” ，匹配一组可选字符

1. “ ? ” 符号

“ ? ” 符号匹配字符 0 次或 1 次。

JO?B
# 匹配 O 字符 0 次或 1 次

2. “ + ” 符号

“ + ” 符号匹配字符 1 次或多次。

S+EU
# 匹配 S 字符 1 次或多次

3. “ () ” 符号

“ () ” 符号表示一个子表达式

(RE1)(RE2)...
# 匹配多个子表达式

# 在 BASH 中，可用变量 ${BASH_REMATCH[]} 表示匹配的子表达式
# ${BASH_REMATCH[0]} = RE
# ${BASH_REMATCH[1]} = RE1
# ${BASH_REMATCH[2]} = RE2

4. “ | ” 符号

“ | ” 符号表示逻辑或

RE1 | RE2 | RE3 |...
