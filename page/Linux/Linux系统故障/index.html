Linux系统故障
白羊
 2020-06-28 09:54:48  
  • CentOS 
 
  • Linux 
 

Linux 出现bash: /usr/bin/*: /lib64/ld-linux-x86-64.so.2: bad ELF interpreter: No such file or directory

误删除glibc-*.x86_64 rpm 包，导致整个系统的大部分命令（cd 命令除外）都不能用

模拟错误，真实环境不要尝试

[root@localhost ~]# rpm -e glibc-* --nodeps
warning: /etc/nsswitch.conf saved as /etc/nsswitch.conf.rpmsave
warning: %postun(glibc-2.17-260.el7.x86_64) scriptlet failed, exit status 127

使用命令系统不识别，并且断开连接后系统也将无法登录

-bash: /usr/bin/ls: /lib64/ld-linux-x86-64.so.2: bad ELF interpreter: No such file or directory
-bash: /usr/bin/cp: /lib64/ld-linux-x86-64.so.2: bad ELF interpreter: No such file or directory
-bash: /usr/bin/mv: /lib64/ld-linux-x86-64.so.2: bad ELF interpreter: No such file or directory

解决办法

关闭主机电源，打开主机电源进入BIOS调整光盘启动（挂载光盘），进入急救模式

关闭主机电源，重启主机（挂载光盘）
进入GRUB引导菜单时按键盘C键，输入EXIT回车
进入光盘引导，选择急救模式，拯救一个CentOS系统，按1回车，输入return获得一个shell

安装删除掉的软件包

sh-4.2# mkdir media
sh-4.2# blkid
sh-4.2# mount /dev/sr0 /media
sh-4.2# mount /dev/sr0 /media
sh-4.2# cd /media/Packages
sh-4.2# rpm -Uvh -r /mnt/sysimage/ glibc*.x86_64.rpm

重启机器

sh-4.2# reboot
