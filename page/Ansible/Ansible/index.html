一、什么是 Ansible？

Ansible 是新出现的自动化运维工具，基于 Python 开发，集合了众多运维工具（puppet、cfengine、chef、func、fabric）的优点，实现了批量系统配置、批量程序部署、批量运行命令等功能。

二、Ansible 的优点？

(1)、轻量级，无需在客户端安装agent，更新时，只需在操作机上进行一次更新即可；
(2)、批量任务执行可以写成脚本，而且不用分发到远程就可以执行；
(3)、使用python编写，维护更简单；
(4)、支持sudo。

Ansible的使用场景

  • 应用代码自动化部署
  • 系统管理配置自动化
  • 支持持续交付自动化

三、Ansible 的实现

Ansible 是基于模块工作的，本身没有批量部署的能力。真正具有批量部署的是 Ansible 所运行的模块，Ansible 只是提供一种框架。主要包括：

(1)、连接插件connection plugins：负责和被监控端实现通信；

(2)、host inventory：指定操作的主机，是一个配置文件里面定义监控的主机；

(3)、各种模块核心模块、command模块、自定义模块；

(4)、借助于插件完成记录日志邮件等功能；

(5)、playbook：剧本执行多个任务时，非必需可以让节点一次性运行多个任务。

Ansible 基础架构

控制端、被控端、inventory、ad-doc、playbook、连接协议。

四、Ansible 的安装

RHEL 或 CentOS 用户安装：配置Epel源

wget -O /etc/yum.repos.d/epel.repo http://mirrors.aliyun.com/repo/epel-7.repo

Yum安装Ansible

yum -y install ansible

查看 Ansible 版本

五、Ansible 的配置

1. Ansible 配置文件

查看 Ansible 版本

$ ansible --version
ansible 2.9.15
  config file = /etc/ansible/ansible.cfg
  configured module search path = [u'/root/.ansible/plugins/modules', u'/usr/share/ansible/plugins/modules']
  ansible python module location = /usr/lib/python2.7/site-packages/ansible
  executable location = /usr/bin/ansible
  python version = 2.7.5 (default, Oct 30 2018, 23:45:53) [GCC 4.8.5 20150623 (Red Hat 4.8.5-36)]
  1. config file：Ansible 配置文件。
  2. configured module search path：配置的模块搜索路径。
  3. ansible python module location：Ansible Python 模块位置。
  4. executable location：可执行位置。
  5. python version：Python 版本。

用户可以修改配置文件来修改设置，配置文件读取顺序：

* ANSIBLE_CONFIG (一个环境变量)
* ansible.cfg (位于当前目录中)
* .ansible.cfg (位于家目录中)
* /etc/ansible/ansible.cfg

版本 1.5 之前的读取顺序：

* ansible.cfg (位于当前目录)
* ANSIBLE_CONFIG (一个环境变量)
* .ansible.cfg (位于家目录下)
* /etc/ansible/ansible.cfg

配置文件的详细参数以及取值范围请查看 ansible.cfg。

2. Inventory 文件

Ansible 可同时操作属于一个组的多台主机，组和主机之间的关系通过 inventory 文件配置.。

默认的文件路径为 /etc/ansible/hosts。

主机与组

mail.example.com

[webservers]
foo.example.com
bar.example.com

[dbservers]
one.example.com
two.example.com
three.example.com

方括号[]中是组名,用于对系统进行分类，便于对不同系统进行个别的管理。

如果有主机的SSH端口不是标准的22端口，可在主机名之后加上端口号，用冒号分隔。

端口号不是默认设置时,可明确的表示为：

badwolf.example.com:5309

配置文件的详细参数以及取值范围请查看 /etc/ansible/hosts。

六、使用 Ansible

1. 创建 Ansible 项目

mkdir /root/ansible
cd /root/ansible
cp /etc/ansible/ansible.cfg /root/ansible/
cp /etc/ansible/hosts /root/ansible/

查看 config file 文件位置为 /root/ansible/ansible.cfg

$ ansible --version
ansible 2.9.17
  config file = /root/ansible/ansible.cfg
  configured module search path = [u'/root/.ansible/plugins/modules', u'/usr/share/ansible/plugins/modules']
  ansible python module location = /usr/lib/python2.7/site-packages/ansible
  executable location = /usr/bin/ansible
  python version = 2.7.5 (default, Oct 30 2018, 23:45:53) [GCC 4.8.5 20150623 (Red Hat 4.8.5-36)]

2. 设置公钥认证

Ansible1.2.1及其之后的版本都会默认启用公钥认证.

控制端管理被控端

复制主机密钥

ssh-keygen
ssh-copy-id -i ~/.ssh/id_rsa.pub user@hostname

取消 SSH 密钥检查

#修改 ansible.cfg 文件
[defaults]
host_key_checking = False

使用 ping 模块

ansible all -m ping -i hosts

3. Ansible Ad-doc

什么是 Ad-doc？

Ad-doc 简而言之就是“临时命令”，命令执行完即结束，并不会保存。

Ad-doc 的使用场景

比如在多台主机上查看某个进程是否启动，或拷贝指定文件到某台主机等。

0）Ansible 语法

usage: ansible [-h] [--version] [-v] [-b] [--become-method BECOME_METHOD]
               [--become-user BECOME_USER] [-K] [-i INVENTORY] [--list-hosts]
               [-l SUBSET] [-P POLL_INTERVAL] [-B SECONDS] [-o] [-t TREE] [-k]
               [--private-key PRIVATE_KEY_FILE] [-u REMOTE_USER]
               [-c CONNECTION] [-T TIMEOUT]
               [--ssh-common-args SSH_COMMON_ARGS]
               [--sftp-extra-args SFTP_EXTRA_ARGS]
               [--scp-extra-args SCP_EXTRA_ARGS]
               [--ssh-extra-args SSH_EXTRA_ARGS] [-C] [--syntax-check] [-D]
               [-e EXTRA_VARS] [--vault-id VAULT_IDS]
               [--ask-vault-pass | --vault-password-file VAULT_PASSWORD_FILES]
               [-f FORKS] [-M MODULE_PATH] [--playbook-dir BASEDIR]
               [-a MODULE_ARGS] [-m MODULE_NAME]
               pattern
ansibleall | 主机名-m command-a ‘df -h’-i hosts
命令目标主机指定模块模块动作INVENTORY

Ansible 命令返回结果

绿色：代表被管理主机没有被修改。

黄色：代表被管理主机发生变更。

红色：代表出现故障，注意查看提示。

1）Ping 模块

Ping 所有主机

ansible all -m ping -i hosts

Ping webservers 组

ansible webservers -m ping -i hosts

2）command 模块

在所有被管理主机上执行 df -h 命令

ansible all -m command -a 'df -h' -i hosts

在command模块中不支持使用变量和管道符

3）shell 模块

在所有被管理主机上执行 df -h 命令

ansible all -m shell -a 'df -h' -i hosts

备份 webservers 组的系统日志

ansible webserves -m shell -a 'cat /var/log/messages > messages.bak' -i hosts

4）yum 模块

安装最新版本的 Apache

ansible webservers -m yum -a 'name=httpd state=latest' -i hosts

卸载 Apache 程序包

ansible webservers -m yum -a 'name=httpd state=absent' -i hosts

安装特定版本的 Apache

ansible webservers -m yum -a 'name=httpd-2.4.6 state=present' -i hosts

在远程仓库安装 Nginx rpm

ansible webservers -m yum -a 'name=http://nginx.org/packages/centos/7/x86_64/RPMS/nginx-1.16.0-1.el7.ngx.x86_64.rpm state=present' -i hosts

从本地文件安装Nginx rpm

ansible webservers -m yum -a 'name=/root/nginx-1.16.0-1.el7.ngx.x86_64.rpm state=present' -i hosts

安装软件包列表

ansible webservers -m yum -a 'name=httpd,wget state=present' -i hosts

升级所有软件包，但不包括与 kernel 和 foo 相关的软件包

ansible webservers -m yum -p 'name=* state=latest exclude=kernel*,foo*' -i hosts

5）copy 模块

复制 httpd.comf 文件到目标主机并备份原始文件

ansible webservers -m copy -a "src=./httpd.conf dest=/etc/httpd/conf/httpd.conf owner=root group=root mode=0644 backup=yes" -i hosts

使用 content 进行复制

ansible webservers -m copy -a "content='# This file was moved to /etc/other.conf' dest=/etc/mime.conf" -i hosts

6）file 模块

更改文件所有权，组和权限

ansible webservers -m file -a "path=/etc/foo.conf owner=foo group=foo mode=0644" -i hosts

给现有文件不安全的权限

ansible webservers -m file -a "path=/work owner=root group=root mode=1777" -i hosts

创建符号链接

ansible webservers -m file -a "src=/file/to/link/to dest=/path/to/symink owner=foo group=foo state=link" -i hosts

创建目录（如果不存在）

ansible webservers -m file -a "path=/etc/some_directory state=directory mode=0755" -i hosts

以递归方式更改目录所有权

ansible webservers -m file -a "path=/etc/foo state=directory recurse=yes owner=foo group=foo" -i hosts

删除文件

ansbile webservers -m file -a "path=/etc/foo.txt state=absent" -i hosts

7）get_url 模块

下载 foo.conf 文件

ansible webservers -m get_url -a "url=http://example.com/path/file.conf dest=/etc/foo.conf mode=0440" -i hosts

带检查的下载文件

ansible webservers -m get_url -a "url=http://example.com/path/file.conf dest=/etc/foo.conf checksum='md5:66dffb5228a211e61d6d7ef4a86f5758'"

8）user group 模块、

group 模块

创建基本组 news，指定 UID = 2020

ansible webservers -m group -a "name=news gid=2020 state=present" -i hosts

创建系统组 http，指定 UID = 8080

ansible webservers -m group -a "name=http gid=8080 system=yes state=present" -i hosts

删除 test 基本组

ansible webservers -m group -a "name=test state=absent" -i hosts

User 模块

添加具有特定 uid 和主要组“ admin”的用户“ johnd”

ansible webservers -m user -a "name=johnd comment='John Doe' uid=1024 group=admin" -i hosts

添加 “lisi” 用户，指定 shell 为 /sbin/nologin，并且不创建家目录

ansible webservers -m user -a "name=lisi shell=/sbin/nologin create_home=no" -i hosts

删除用户“ johnd”，同时删除家目录

ansible webservers -m user -a "name=johnd state=absent remove=yes" -i hosts

在 ~/.ssh/id_rsa 中为用户 jsmith 创建一个2048位SSH密钥

ansible webservers -m user -a "name=jsmith generate_ssh_key=yes ssh_key_bits=2048 ssh_key_file=.ssh/id_rsa" -i hosts

创建用户时指定密码

#将密码加密
ansible localhost -m debug -a "msg={{ '1234.com' | password_hash('sha512','salt') }}"

输出结果如下：

localhost | SUCCESS => {
    "msg": "$6$salt$aIhSzbcqnHao8UEpRvPAVbpVyoIZi7a8LzBMq9OpySpEg68crzwHG17vPMbfpyp56e2rhiSanRN7WC9xX1cUP."
}
#创建用户时指定密码
ansible webservers -m user -a 'name=aaa password=$6$salt$aIhSzbcqnHao8UEpRvPAVbpVyoIZi7a8LzBMq9OpySpEg68crzwHG17vPMbfpyp56e2rhiSanRN7WC9xX1cUP.' -i hosts

注意使用单引号，或者添加转义字符。

9）service systemd 模块

Service 模块

启动服务httpd（如果未启动）

ansible webservers -m service -a "name=httpd state=started" -i hosts

停止服务httpd（如果已启动）

ansible webservers -m service -a "name=httpd state=stopped" -i hosts

在所有情况下都重新启动服务httpd

ansible webservers -m service -a "name=httpd state=restarted" -i hosts

在所有情况下都重新加载服务httpd

ansible webservers -m service -a "name=httpd state=reloaded" -i hosts

启用服务httpd，并且开机自启

ansible webservers -m service -a "name=httpd enabled=yes" -i hosts

重新启动接口eth0的网络服务

ansible webservers -m service -a "name=network state=restarted args=eth0" -i hosts

Systemd 模块

确保 httpd 服务正在运行

ansible webservers -m systemd -a "state=started name=httpd" -i hosts

在所有情况下，在 CentOS 上重新启动服务 cron，还发出 daemon-reload 来获取配置更改

ansible webservers -m systemd -a "state=restarted daemon_reload=yes name=crond" -i hosts

在所有情况下都重新加载服务httpd

ansible webservers -m systemd -a "name=httpd state=reloaded" -i hosts

只是强制系统重新读取配置（2.4及更高版本）

ansible webservers -m systemd -a "daemon_reload=yes" -i hosts

10）firewalld selinux 模块

Firewalld 模块

允许 https 的流量通过，下次重启策略自动加载

ansible webservers -m firewalld -a "service=https permanent=yes state=enabled" -i hosts

允许 8081/tcp 的流量通过，且策略状态为禁用，下次重启策略自动加载

ansible webservers -m firewalld -a "port=8081/tcp permanent=yes state=disabled" -i hosts

SELinux 模块

启用 SELinux

ansible webservers -m selinux -a "policy=targeted state=enforcing" -i hosts

将 SELinux 置于宽松模式，记录将被阻止的操作。

ansible webservers -m selinux -a "policy=targeted state=permissive" -i hosts

禁用 SELinux

ansible webservers -m selinux -a "state=disabled" -i hosts

11）mount 模块

ansible 主机组 -m mount -a "src=设备路径 path=挂载点 fstype=文件系统 state=挂载类型"
参数
path		//挂载点
src			//需要挂载的设备
fstype		//挂载设备的文件系统
	iso9660	//光驱
	ext4、xfs、nfs
	cifs samba的共享文件系统
	ntfs windows磁盘文件系统
opts		//挂载属性
	notime
	noexec
	nosuid
state		//挂载动作
	present		//开机挂载，仅将挂载配置写入/etc/fstab并不会真的挂载
	mounted		//挂载设备，并将配置写入/etc/fstab
	unmounted		//卸载设备，不会清除/etc/fstab写入的配置
	absent		//卸载设备，并清理/etc/fstab写入的配置

12）cron 模块

添加定时任务，每分钟执行一次

ansible webservers -m cron -a "name=job1 job='ls > /dev/null'" -i hosts

添加定时任务，每天2点，5点执行一次

ansible webservers -m cron -a "name=job2 minute=0 hour=5,2 job='ls > /dev/null'" -i hosts

关闭定时任务

ansible webservers -m cron -a "name=job1 job='ls > /dev/null' disabled=yes" -i hosts

minute,hour,day,month

13）unarchive模块

将 foo.tgz 解压缩到 /var/lib/foo

ansible webservers -m unarchive -a "src=foo.tgz dest=/var/lib/foo" -i hosts

4. Ansible Playbook

1) Playbook 语法

usage: ansible-playbook [-h] [--version] [-v] [-k]
                        [--private-key PRIVATE_KEY_FILE] [-u REMOTE_USER]
                        [-c CONNECTION] [-T TIMEOUT]
                        [--ssh-common-args SSH_COMMON_ARGS]
                        [--sftp-extra-args SFTP_EXTRA_ARGS]
                        [--scp-extra-args SCP_EXTRA_ARGS]
                        [--ssh-extra-args SSH_EXTRA_ARGS] [--force-handlers]
                        [--flush-cache] [-b] [--become-method BECOME_METHOD]
                        [--become-user BECOME_USER] [-K] [-t TAGS]
                        [--skip-tags SKIP_TAGS] [-C] [--syntax-check] [-D]
                        [-i INVENTORY] [--list-hosts] [-l SUBSET]
                        [-e EXTRA_VARS] [--vault-id VAULT_IDS]
                        [--ask-vault-pass | --vault-password-file VAULT_PASSWORD_FILES]
                        [-f FORKS] [-M MODULE_PATH] [--list-tasks]
                        [--list-tags] [--step] [--start-at-task START_AT_TASK]
                        playbook [playbook ...]

2）搭建 httpd 服务

编写 httpd.yml 文件：

- hosts: webservers
  tasks:
    - name: Install httpd Server
      yum:
        name: httpd
        state: latest

    - name: Copy index.html File.
      copy:
        src: ./index.html
        dest: /var/www/html/
        owner: root
        group: root
        mode: '0644'

    - name: Service httpd Server.
      service:
        name: httpd
        state: started
        enabled: yes

    - name: Configure firewall.
      firewalld:
        port: 80/tcp
        permanent: yes
        state: enabled
        immediate: yes

运行 ansible-playbook 命令

ansible-playbook httpd.yml -i hosts
