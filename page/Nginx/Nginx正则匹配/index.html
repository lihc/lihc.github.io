Nginx 正则匹配

  1. 以 = 开头，表示精确匹配
  2. 以 ^~ 开头，表示uri以某个常规字符串开头，理解为匹配 URL 路径即可
  3. ~ 开头，表示区分大小写的正则匹配
  4. ~* 开头，表示不区分大小写的正则匹配
  5. !~ 和!~* 分别为区分大小写不匹配及不区分大小写不匹配 的正则
  6. / 通用匹配，任何请求都会匹配到。

一、精确匹配

以=开头，表示精确匹配

location = / {
    root    /usr/local/nginx/html;
    index   index.html index.htm;
}
location = /a.png {
    root    /usr/local/nginx/html;
    index   index.html index.htm;
}

访问127.0.0.1和127.0.0.1/a.png精准匹配。

二、普通匹配

/ 通用匹配，任何请求都会匹配到。

location / {
    root   html;
    index  index.html index.htm;
}

三、正则匹配

1. 以^~开头，表示 URL 以某个字符串开头，理解为匹配 URL 路径即可

location ^~ /static {
    root    /usr/local/nginx/html/;
    index   index.html,index.htm;
}

访问127.0.0.1/static/正则匹配。

2. 以~ 开头，表示区分大小写的正则匹配

location ~ \.(gif|jpg|jpeg|png|bmp|swf|js|css)$ {
    root    /usr/local/nginx/html/;
    index   index.html,index.htm;
}

3. 以~* 开头，表示不区分大小写的正则匹配

location ~* \.png$ {
    root    /usr/local/nginx/html/;
    index   index.html,index.htm;
}

4. 以!~ 和!~* 分别为区分大小写不匹配及不区分大小写不匹配 的正则匹配

location !~ \.(gif|jpg|jpeg|png|bmp|swf|js|css)$ {
    root    /usr/local/nginx/html/;
    index   index.html,index.htm;
}
location !~* \.png$ {
    root    /usr/local/nginx/html/;
    index   index.html,index.htm;
}

四、匹配顺序

location 命中过程是这样的：

  1. 先判断精准命中，如果命中，立即返回结果并结束解析过程

  2. 判断普通命中，如果有多个命中，记录下来最长的命中结果

  3. 继续判断正则表达式的解析结果，按配置里的正则表达式顺序为准，由上到下开始匹配，一旦匹配成功立刻返回结果，并结束解析过程。

延伸分析：

a. 普通命中：顺序无所谓，是因为按命中长短来确定的

b. 正则命中：顺序有所谓，因为是从前往后命中的

root 和 alias 的区别

# http://127.0.0.1/static
# location /static { 则没有区别
location /static/ {
    # 访问目录:var/www/html/static/下的文件
    # root /var/www/html;
    
    # 访问目录:var/www/html/static/下的文件
    # root /var/www/html/;
    
    # 会出现 403 错误,配置错误
    alias /var/www/html;
    
    # 访问目录:var/www/html/下的文件
    # alias /var/www/html/;
    
    index index.html index.htm;
}

location+root+/static/
