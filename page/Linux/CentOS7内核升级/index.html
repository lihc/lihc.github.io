CentOS 7内核升级
白羊
 2020-10-15 09:18:30  
  • Linux 
 
  • CentOS 
 

CentOS 7.6 默认的内核版本为 3.10，当 3.10 版本的内核不适用时，就需要对内核进行升级。

关于内核版本的定义：

版本性质：主分支ml(mainline)，稳定版(stable)，长期维护版lt(longterm)

版本命名格式为 “A.B.C”：

数字 A 是内核版本号：版本号只有在代码和内核的概念有重大改变的时候才可能会改变。

数字 B 是内核主版本号：主版本号根据传统的奇-偶系统版本编号来分配：奇数为开发版，偶数为稳定版。

数字 C 是内核次版本号：次版本号是无论在内核增加安全补丁、修复bug、实现新的特性或者驱动时都会改变。

1. 查看当前内核版本

$ uname -r
3.10.0-957.el7.x86_64uname -a
$ uname -a
Linux master 3.10.0-957.el7.x86_64 #1 SMP Thu Nov 8 23:39:32 UTC 2018 x86_64 x86_64 x86_64 GNU/Linux
$ cat /etc/redhat-release
CentOS Linux release 7.6.1810 (Core)

2. 启用 ELRepo仓库

导入ELRepo仓库的公共密钥

rpm --import https://www.elrepo.org/RPM-GPG-KEY-elrepo.org

安装ELRepo仓库的yum源

rpm -Uvh http://www.elrepo.org/elrepo-release-7.0-2.el7.elrepo.noarch.rpm

3. 对内核进行升级

安装最新主线稳定版本内核

yum -y --enablerepo=elrepo-kernel install kernel-ml.x86_64 kernel-ml-devel.x86_64

查看系统可用内核

$ awk -F\' '$1=="menuentry " {print $2}' /etc/grub2.cfg
CentOS Linux (5.9.2-1.el7.elrepo.x86_64) 7 (Core)
CentOS Linux (3.10.0-957.el7.x86_64) 7 (Core)
CentOS Linux (0-rescue-22ef058c04544c77b9d190be469fd5eb) 7 (Core)

当前系统默认启动内核

$ grub2-editenv list
saved_entry=CentOS Linux (3.10.0-957.el7.x86_64) 7 (Core)

修改grub中默认的内核版本

grub2-set-default "CentOS Linux (5.9.2-1.el7.elrepo.x86_64) 7 (Core)"

查看当前内核启动项

grub2-editenv list
saved_entry=CentOS Linux (5.9.2-1.el7.elrepo.x86_64) 7 (Core)

重启系统使内核生效

$ reboot
$ uname -r
5.9.2-1.el7.elrepo.x86_64
