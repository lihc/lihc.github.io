安装Apache服务
白羊
 2020-06-30 14:09:43  
  • Linux 
 
  • CentOS 
  • | Apache 
 

前置准备工作

查看系统是否安装httpd服务

[root@localhost ~]# rpm -e httpd --nodeps
error: package httpd is not installed

Apache 下载地址：http://httpd.apache.org/download.cgi

下载Apache源码包

[root@localhost ~]# wget https://mirrors.tuna.tsinghua.edu.cn/apache//httpd/httpd-2.4.43.tar.gz
因为Apache网站会不定时更新httpd的版本，所以链接不是永久有效

安装所需依赖包

[root@localhost ~]# yum -y install gcc-c++
[root@localhost ~]# yum -y install apr apr-deve cyrus-sasl-devel expat-devel libdb-devel openldap-devel apr-util-devel apr-util pcre-devel pcre

安装httpd服务

解压源码包，进行编译安装

[root@localhost ~]# tar zxf httpd-2.4.43.tar.gz -C /usr/src/
[root@localhost ~]# cd /usr/src/httpd-2.4.43/
[root@localhost httpd-2.4.43]# ./configure --prefix=/usr/local/httpd --enable-so --enable-rewrite --enable-charset-lite --enable-cgi && make && make install

编译参数说明

--prefix 指定httpd服务程序的安装目录
--enable-so 启用动态加载模块支持，使httpd具备进一步扩展功能的能力
--enable-rewrite 启用网页地址重写，用于网站优化及目录迁移维护
--enable-charset-lite 启动字符集支持，以便支持使用各种字符集编码的网页
--enable-cgi 启用CGI脚本程序支持，便于扩展网站的应用访问能力
--enable-ssl 编译SSL模块
--with-zlib 支持ZLIB压缩
--with-pcre 支持正则表达式
--enable-mpms-shared=all 以共享方式编译的模块
--with-mpm=prefork 指明httpd的工作方式为prefork

验证安装目录

[root@localhost ~]# ls /usr/local/httpd/
bin  build  cgi-bin  conf  error  htdocs  icons  include  logs  man  manual  modules

优化执行路径

[root@localhost ~]# ln -s /usr/local/httpd/bin/* /usr/local/bin/
[root@localhost ~]# ls -l /usr/local/bin/httpd /usr/local/bin/apachectl
lrwxrwxrwx 1 root root 30 Jun  5 08:40 /usr/local/bin/apachectl -> /usr/local/httpd/bin/apachectl
lrwxrwxrwx 1 root root 26 Jun  5 08:40 /usr/local/bin/httpd -> /usr/local/httpd/bin/httpd

PS：这样当执行httpd -v命令时就相当于执行了/usr/local/bin/httpd -v命令

添加系统服务

将apachectl脚本复制为/etc/init.d/httpd,并在文件开头添加chkconfig识别配置，在/etc/init.d/httpd文件中添加两行

[root@localhost ~]# cp /usr/local/httpd/bin/apachectl /etc/init.d/httpd
[root@localhost ~]# vim /etc/init.d/httpd
#!/bin/sh
#
# chkconfig:35 85 21	//服务识别参数，在级别3、5中启动，启动和关闭顺序分别为85、21
# description:Startup script for the Apache HTTP Server		//服务描述信息
...

使用chkconfig添加系统服务，然后由systemctl进行管理

[root@localhost ~]# chkconfig --add httpd

建立[service].service配置文件

配置文件主要分三部分，控制单元[Unit]的定义、服务[Service]的定义以及安装部分[Install]
[root@localhost ~]# vim /lib/systemd/system/httpd.service
[Unit]
Description=The Apache HTTP Server
After=network.target

[Service]
Type=forking
PIDFile=/usr/local/httpd/logs/httpd.pid
ExecStart=/usr/local/bin/apachectl $OPTIONS
ExecReload=/bin/kill -HUP $MAINPID
KillMode=process
Restart=on-failure
RestartSec=42s

[Install]
WantedBy=graphical.target

使用systemctl设置httpd服务开机自启

[root@localhost ~]# systemctl start httpd.service
[root@localhost ~]# systemctl enable httpd.service
Created symlink from /etc/systemd/system/graphical.target.wants/httpd.service to /usr/lib/systemd/system/httpd.service.

查看httpd端口

[root@localhost ~]# netstat -anpt | grep httpd
tcp6    0   0 :::80          :::*          LISTEN   23908/httpd

本地验证：访问网页http://127.0.0.1:80得到结果 It works!

[root@localhost ~]# curl 127.0.0.1:80
<html><body><h1>It works!</h1></body></html>

在安装后的/usr/local/httpd目录下，主要子目录用途如下

  • /usr/local/httpd/bin：存放httpd服务各种执行程序文件，主程序httpd、服务控制工具apachectl等
  • /usr/local/httpd/cgi-bin：存放各种CGI程序文件
  • /usr/local/httpd/logs：存放httpd服务的日志文件
  • /usr/local/httpd/conf：存放httpd服务的各种配置文件，包括主配置文件httpd.conf
  • /usr/local/httpd/htdocs：存放网页文档，包括首页文件index.html等
  • /usr/loca/httpd/modules：存放httpd服务的各种模块文件

补充：未添加httpd系统服务时的Apache启动命令

[root@localhost ~]# /usr/local/httpd/bin/apachectl start     启动服务
[root@localhost ~]# /usr/local/httpd/bin/apachectl restart   重启服务
[root@localhost ~]# /usr/local/httpd/bin/apachectl stop     停止服务
\#其中/usr/local/应为Apache的安装目录
