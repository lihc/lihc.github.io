文件系统类型

  • ext：Linux 中最早的文件系统，由于在性能和兼容性上具有很多缺陷，现在已经很少使用。

  • ext2：Red Hat 7.2 版本以前的系统默认都是 Ext2 文件系统。于 1993 年发布，支持最大 16TB 的分区和最大 2TB 的文件。

  • ext3：增加日志功能，可回溯追踪，以便在系统突然停止时提高文件系统的可靠性。支持最大 16TB 的分区和最大 2TB 的文件。

  • ext4：日志式文件系统，Ext4 在性能、伸缩性和可靠性方面进行了大量改进。

  • xfs：被业界称为先进的、最具可升级性的文件系统技术。

fstab 文件字段含义：

  • file systems：挂载设备，设备（分区）的名字，也可以使用“UUID=”。
  • mount point：挂载点，设备挂载到的文件目录。
  • type： 文件系统类型，( ext2 | ext3 | ext4 | xfs … ) 。
  • options：文件系统参数，(defaults | rw | ro ) ……
  • dump：( 0 | 1 ) ，对于 dump 决定是否备份。0 表示忽略。
  • pass：( 0 | 1 | 2 )，文件系统的检查 ( fsck ) 顺序，含义代表为 “忽略检查”、”优先检查” 和 “需要检查”。
