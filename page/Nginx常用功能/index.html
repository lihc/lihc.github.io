Nginx 常用功能

一、Nginx信号控制

Nginx 可以通过信号进行控制。默认情况下，主进程 ID 被写入 /usr/local/nginx/logs/nginx.pid文件。

主进程支持以下信号：

信号 说明
TERM, INT fast shutdown #快速关闭
QUIT graceful shutdown #正常关闭
HUP changing configuration。 #平滑的重读配置文件
USR1 re-opening log files #重读日志文件
USR2 upgrading an executable file #升级可执行文件(平滑的升级)
WINCH graceful shutdown of worker processes #正常关闭工作进程(配合USR2进行升级)

二、静态网站部署

查看配置文件（省略部分内容）

  • server：代表一个虚拟主机。
  • listen：监听端口。
  • server_name：主机名称。
  • location：主机配置。
  • root：网站目录，html目录在/usr/local/nginx/下。
  • index：首页文件。

将静态网页放在/usr/local/nginx/html下，访问主机 80 端口即可。

三、配置虚拟主机

虚拟主机，也叫“网站空间”，就是把一台运行在互联网上的物理服务器划分成多个“虚拟”服务器。虚拟主机技术极大的促进了网络技术的应用和普及。

划分虚拟主机的三种方法：

  • 基于端口划分虚拟主机
  • 基于域名划分虚拟主机
  • 基于 IP 地址划分虚拟主机

五、Nginx正则匹配

  1. 以 = 开头，表示精确匹配
  2. 以 ^~ 开头，表示uri以某个常规字符串开头，理解为匹配 URL 路径即可
  3. ~ 开头，表示区分大小写的正则匹配
  4. ~* 开头，表示不区分大小写的正则匹配
  5. !~ 和!~* 分别为区分大小写不匹配及不区分大小写不匹配 的正则
  6. / 通用匹配，任何请求都会匹配到。

十、负载均衡

upstream 模块：http://nginx.org/en/docs/http/ngx_http_upstream_module.html

  • server Host:Port：后端服务器的IP和端口列表配置
  • weight：权重轮询调度
  • down：该主机暂停服务
  • max_fails：失败最大次数，超过失败最大次数暂停服务
  • fail_timeout：如果请求受理失败，暂停指定的时间之后重新发起请求
  • backup：备份服务器，当后端服务器全部Down后提供服务

十一、目录索引

语法 默认 语境 说明

autoindex_exact_size on | off; on http，server，location 启用或禁用目录列表输出
autoindex_localtime on| off; off http，server，location 是否应在本地输出目录列表中的时间
autoindex_format html | xml | json | jsonp; html http，server，location 设置目录列表的格式
