12.    VLAN 的范围：
	  VLAN ID		范围			用途
	  0、4095		保留		仅限系统使用。用户不能查看和使用
	        1		正常		Cisco 默认的 VLAN。用户可以使用，但不能删除
	    2-1001		正常		用于以太网的 VLAN。用户可以创建、使用和删除
	1002-1005	正常		用于 FDDI 和令牌环的 Cisco 默认 VLAN。用户不能删除
	1006-1024	保留		仅限系统使用。用户不能查看和使用
	1025-4094	扩展		仅用于以太网 VLAN

       创建 VLAN：
	Switch#vlan database 	// 进入 VLAN 数据库
	Switch(vlan)#vlan 2		// 创建 VLAN 2
	VLAN 2 added:
	    Name: VLAN0002		// VLAN 名字
	Switch(vlan)#vlan 3 name test		// 指定 VLAN3 名字为 test
	Switch(vlan)#		// 退出 exit

       删除 VLAN：
	Switch#vlan database
	Switch(vlan)#no vlan 3
	Deleting VLAN 3...
	Switch(vlan)#

       交换机端口加入 VLAN：
	Switch(config)#interface fastEthernet 0/1

	Switch(config-if)#
		// 连续接口可以使用Switch(config)#interface range fastEthernet 0/1 - 9

       查看 VLAN 信息：（三种）

2.    STP 简介：
	STP（Spanning Tree Protocol，生成树协议）把一个环形的结构改变成一个树形的结构

       STP 的作用：
	逻辑上断开环路，防止广播风暴的产生
	当线路故障，阻塞接口被激活，恢复通信，起到备份线路的作用

       生成树算法：STA（Spanning Tree Algorithm）
	（1）选择根网桥（Root Bridge）
	（2）选择跟端口（Root Ports）
	（3）选择指定端口（Designated Ports）

       选择根网桥的依据：
	交换网络中网桥ID最小的交换机成为根网桥

       选择根端口的依据：（在非根网桥上选择）
	（1）到根网桥最低的根路径成本
	（2）直连的网桥ID最小
	（3）直连端口ID最小

       选择指定端口的依据:
	1、根网桥上的端口全是指定端口
	2、在每个链路上，选择1个指定端口
	3、非根桥上的指定端口，选择顺序：
	（1）根路径成本较低
	（2）所在的交换机的网桥ID的值较小
	（3）所在端口ID的值较小

       交换机端口的STP状态：
	转发，学习，侦听，阻塞，禁用

       PVST+ 配置命令
       启用生成树：
	Switch(config)#spanning-tree vlan VLAN ID号	// 默认生成树启用，关闭命令前加 NO

       修改端口成本：
	Switch(config-if)#spanning-tree vlan VLAN ID号 cost 成本

       修改端口优先级：
	Switch(config-if)#spanning-tree vlan VLAN ID号 port-priority 优先级

       指定根网桥：
	配置主根
	Switch(config)#spanning-tree vlan VLAN ID号 root primary
	配置副根（备份）
	Switch(config)#spanning-tree vlan VLAN ID号 root secondary

       配置速端口：
	Switch(config)#interface F0/1	
	Switch(config-if)#spanning-tree portfast
	Switch(config-if)#
		// 在交换机和计算机相连的接口上配置

       查看生成树的配置：
	Switch#show spanning-tree

       查看某个 VLAN 的生成树配置：
	Switch#show spanning-tree vlan VLAN ID号

       桥协议数据单元
       BPDU 的两种类型：
	配置 BPDU，用于生成树计算
	TCN（Topology Change Notification，拓扑变更通告）BPDU，用于通告网络拓扑的变化


3.    HSRP（Hot Standby Routing Protocol，热备份路由选择协议）

       热备份路由选择协议(HSRP)的作用：用来对网关的备份（冗余）

       活跃路由器上的配置（VLAN2 为例）
	Switch(config)#interface vlan 2
	Switch(config-if)#ip address 192.168.1.1 255.255.255.0	// 配置虚接口 IP 地址
	Switch(config-if)#no shutdown
	Switch(config-if)#standby 2 ip 192.168.1.254		// 虚拟路由 IP 地址，同时为 VLAN 内主机的网关 IP 地址
	Switch(config-if)#standby 2 priority 200			// 优先级为 200，默认值 100，范围是 0 - 255
	Switch(config-if)#standby 2 preempt			// 占先权，路由器优先级高的从优先级低的取回转发权
	Switch(config-if)#standby 2 track F0/0 100		// 跟踪端口 F0/0，当 F0/0 关闭时，优先级将降低 100
	Switch(config-if)#
       备份路由器上的配置（VLAN2 为例）
	Switch2(config)#interface vlan 2
	Switch2(config-if)#ip address 192.168.1.2 255.255.255.0
	Switch2(config-if)#no shutdown
	Switch2(config-if)#standby 2 ip 192.168.1.254		// 虚拟路由 IP 地址
	Switch2(config-if)#standby 2 priority 150		// 优先级为150
	Switch2(config-if)#standby 2 preempt			// 配置占先权
	Switch2(config-if)#
		// standby 后面的 2 为 VLAN ID 号

       查看HSRP的信息：
	Switch#show standby brief

       HSRP 的状态：
	初始状态，学习状态，监听状态，发言状态，备份状态，活跃状态


4.    IP 地址的分类：
	经过子网划分之后，IP 地址可以分为 有类地址 和 无类地址
	有类地址：标准的 IP 地址（A、B、C 三类）属于有类地址
	无类地址：进行子网划分后的 IP 地址不再具有有类地址特征，称为无类地址

	IP 地址由两部分组成：网络部分（net ID）和主机部分（host ID）
	网络部分用于表示一个网络中的特定主机

5.    ACL（Access Control List，访问控制列表）的类型：
	标准 ACL：
		根据数据包的源 IP 地址来允许或拒绝数据包，访问控制列表号是 1 - 99
	扩展 ACL：
		根据数据包的源 IP 地址、目的 IP 地址、指定协议、端口和标志来允许或拒绝数据包，访问控制列表号是 100 - 199
	命名 ACL：
		命名 ACL 允许在标准 ACL 和扩展 ACL 中使用名称代替表号

       标准 ACL 的配置：
       允许或拒绝一个网段(192.168.1.0/24)
	Router(config)#access-list 2 permit 192.168.1.0 0.0.0.255	// permit 允许网段，子网掩码使用反码
              或
	Router(config)#access-list 2 deny) 192.168.1.0 0.0.0.255	// deny 拒绝网段

       允许或拒绝一个主机(192.168.1.100/24)
	Router(config)#access-list 2(表号1-99) permit(或deny) host 192.168.1.100
              或
	Router(config)#access-list 2(表号1-99) permit(或deny) 192.168.1.100 0.0.0.0

       允许或拒绝所有的主机
	Router(config)#access-list 2(表号1-99) permit(或deny) any
             或
	Router(config)#access-list 2(表号1-99) permit(或deny) 0.0.0.0 255.255.255.255

       应用到接口上（F0/1）
	Router(config)#interface F0/1
	Router(config-if)#ip access-group 2 in | out	// in 入站接口，out 出站接口
	Router(config-if)#

       查看 ACL：
	Router#show access-lists

       扩展 ACL 的配置：
       例1.    允许网络 192.168.1.0/24 访问网络192.168.2.0/24 的 IP 流量通过，而拒绝其他任何流量
	Router(config)#access-list 101 permit ip 192.168.1.0 0.0.0.255 192.168.2.0 0.0.0.255
	Router(config)#access-list 101 deny ip any any
       例2.    拒绝网络 192.168.1.0/24 访问 FTP 服务器 192.168.2.2/24 的 IP 流量通过，而允许其他任何流量
	Router(config)#access-list 101 deny tcp 192.168.1.0 0.0.0.255 host 192.168.2.2 eq 21
	Router(config)#access-list 101 permit ip any any
       例3.    禁止网络 192.168.1.0/24 中的主机 ping 通服务器 192.168.2.2/24 ，而允许其他任何流量
	Router(config)#access-list 101 deny icmp 192.168.1.0 0.0.0.255 host 192.168.2.2 echo
	Router(config)#access-list 101 permit ip any any
		// 扩展 ACL 较为复杂，具体内容不在详细整理

6.    网络地址转换 NAT（Network Address Translation）

       NAT 相关术语：
	内部局部 IP 地址（Inside Local IP Address）：在内部网络中分配给主机的私有 IP 地址
	内部全局 IP 地址（Inside Global IP Address）：一个合法的 IP 地址（由 NIC 或网络服务提供商分配）
	外部全局 IP 地址（Outside Global IP Address）：由所有者给外部网络上的主机分配的 IP 地址
	外部局部 IP 地址（Outside Local IP Address）：外部主机表现在内部网络的 IP 地址
	简单转换条目（Simple Translation Entry）：将一个 IP 地址映射到另一个 IP 地址的转换条目
	扩展转换条目（Extended Translation Entry）：映射 IP 地址和端口到另一对 IP 地址和端口的条目

       NAT 实现方式：
	静态转换（Static Translation）：主要实现内部服务器发布，采用一对一，或端口映射
	动态转换（Dynamic Translation）：主要实现内网访问外网，采用 多对多,不常用
	PAT（Port-Base Address Translation，基于端口的地址转换）：主要实现内网访问外网，采用多对一，通过端口来区分

       静态 NAT：
	Router(config)#ip nat inside source static 本地地址  全局地址

	Router(config)#interface F0/0		// 内部接口
	Router(config-if)#ip nat inside 
	Router(config-if)#

	Router(config)#interface F0/1		// 外部接口
	Router(config-if)#ip nat outside 
	Router(config-if)#

       动态 NAT：
	定义一个全局地址池	// 可以重复此命令添加地址池
	Router(config)#ip  nat  pool  名字  开始IP  结束IP  netmask  掩码

	定义内部访问列表
	Router(config)#access-list  表号  premit  源地址  反码

	利用事先建立的ACL表建立动态的源地址转换
	Router(config)#ip  nat  inside  source  list  表号  pool  名字

	Router(config)#interface F0/0		// 内部接口
	Router(config-if)#ip nat inside 
	Router(config-if)#

	Router(config)#interface F1/0		// 外部接口
	Router(config-if)#ip nat outside 
	Router(config-if)#

       PAT 多转一：
	定义一个全局地址池
	Router(config)#ip  nat  pool  名字  开始IP  结束IP  netmask  掩码	// 开始 IP 与结束 IP 相同

	定义内部访问控制列表
	Router(config)#access-list  表号  premit  源网段  反码

	复用动态 IP 地址转换
	Router(config)#ip  nat  inside  source  list  表号  interface F1/0 overload

	Router(config)#interface F0/0		// 内部接口
	Router(config-if)#ip nat inside 
	Router(config-if)#

	Router(config)#interface F1/0		// 外部接口
	Router(config-if)#ip nat outside 
	Router(config-if)#

       PAT 多转多：
	定义一个全局地址池
	Router(config)#ip nat pool 名字 小IP 大IP netmask 掩码	// 如果大小 IP 相同，则为 PAT 多转一

	定义内部访问控制列表
	Router(config)#access-list  表号  premit  源网段  反码

	复用动态 IP 地址转换
	Router(config)#ip  nat  inside  source  list  表号 pool 名字 overload

	Router(config)#interface F0/0		// 内部接口
	Router(config-if)#ip nat inside 
	Router(config-if)#

	Router(config)#interface F1/0		// 外部接口
	Router(config-if)#ip nat outside 
	Router(config-if)#

       清除所有动态地址转换条目：
	Router#clear  ip  nat  translation  *	// * 号为通配符

       跟踪 NAT 的操作：
	Router#debug ip nat

       检查当前存在的转换
	Router#show  ip nat  translations

       显示转换统计：
	Router#show  ip  nat  statistics

1.    RIP 路由协议（跳数）：
	RIP 版本 1
	Router(config)#router rip		// 启用 RIP 进程
	Router(config-router)#network 192.168.1.0	// 宣告主网络号
	Router(config-router)#

	RIP 版本 2		// 常用版本
	Router(config)#router rip
	Router(config-router)#version 2	// 启用版本 2
	Router(config-router)#no auto-summary 	// 关闭路由汇总
	Router(config-router)#network 192.168.1.0
	Router(config-router)#

       查看路由表：
	Router#show ip route

       查看路由协议配置：
	Router#show ip protocols

       RIP 版本区别：
	RIP v1
	有类路由，广播更新(255.255.255.255)，不支持 VLSM(可变长子网掩码)，自动路由汇总，不可关闭，不支持不连续子网，能学习到 v1 和 v2
	RIP v2
	无类路由，组播更新(224.0.0.9)，支持 VLSM，自动路由汇总可关闭，可手工汇总，支持不连续子网，只能学习到 v2

2.    OSPF 路由协议（cosp 代价）：
	Router(config)#router ospf  进程号
	Router(config-router)#router-id  ID号
	Router(config-router)#network  网络地址  反掩码  area  区域号
	Router(config-router)#

       查看邻居关系：
	Router#show ip ospf neighbor

       修改接口优先级：
	Router(config-if)#ip ospf priority 101		// 范围是0 - 255

       修改接口 Cost 值：
	Router(config-if)#ip ospf cost 1000

       查看 OSPF 的配置：
	Router#show ip ospf

       OSPF 协议需要保存三张表：
	邻居列表
	链路状态数据库（LSDB）
	路由表

       OSPF 的特点：
	可适应大规模网络
	路由变化收敛速度快
	无路由环
	支持可变长子网掩码（VLSM）
	支持区域划分
	支持以组播地址发送协议报文

       OSPF 的工作过程：
	建立邻接关系——链路状态数据库(Dijkstra 算法)——最短路径树——路由表——学习链路状态信息

       Router-ID 选取规则：
	（1）选取路由器 LoopBack 接口上数值最高的 IP 地址
	（2）如果没有 LoopBack 接口，在特定端口中选取 IP 地址最高的
	（3）也可以使用 router-id 命令指定 Router-ID

       DR 和 BDR 的选举：
	自动选举 DR 和 BDR，网段中 Router-ID 最大的为 DR，第二大为 BDR
	手工选举，优先级范围 0 - 255，数值越大，优先级越高，默认为 1，优先级相同，比较 Router-ID，优先级设置为 0，则不参加选举
	如果 DR 和 BDR 已经存在，不管什么情况，都不在进行选举

       OSPF 的组播地址：
	区域内：224.0.0.5
	区域间：224.0.0.6

3.    生成 OSPF 多区域的原因：
	改善网络的可扩展性，快速收敛

       区域相关的通信量类型：
	域内通信量（Intra-Area Traffic）
	域间通信量（Inter-Area Traffic）
	外部通信量（External Traffic）

       分层路由的优势：
	降低了 SPF 运算的频率
	减小了路由表
	减小了链路状态更新报文（LSU）的流量

       路由器的类型：
	内部路由器（Internal Router）
	区域边界路由器（Area Border Router，ABR）
	自治系统边界路由器（Autonomous System Boundary Router，ASBR）

       区域的类型：
	骨干区域 Area 0：该区域 ID 一定是 0，它连接所有其他区域的核心域，相当与交换网络的汇聚层
	标准区域：该区域可以接收各种链路状态信息和汇总的路由通告
	末梢区域（stub），完全末梢区域，非纯末梢区域（nssa），完全非纯末梢区域

       各种通信量说明：
	LSA 1	路由器 LSA	由区域内路由器发出的
	LSA 2	网络 LSA		由区域内的 DR 发出的
	LSA 3	网络汇总 LSA	ABR 发出的，其他区域的汇总链路通告
	LSA 4	ASBR 汇总 LSA	ABR 发出的，用于通告 ASBR 信息
	LSA 5	AS 外部 LSA	ASBR 发出的，用于通告外部路由
	LSA 7	NSSA 外部 LSA	NSSA 区域内的 ASBR 发出的，用于通告本区域连接的外部路由

       划分末梢的目的：为了减轻非骨干区域路由器的负载

       OSPF通信量分为域内（LSA1，2）,域间（LSA3）,外部(LSA4，5)，NSSA外部(LSA7)

       标准区域允许域内，域间，外部;LSA为1,2,3,4,5
       各种末梢：
	普通末梢（stub）:不允许外部通信量，包括NSSA外部，允许的LSA为1,2,3及一条默认路由。
	完全末梢：只允许域内通信量及一条默认路由，允许的LSA为1,2
	非纯末梢（NSSA）：不允许其他区域的外部通信量，允许的LSA为1,2,3，7
	完全非纯末梢：只允许本区域内部，本区域外部通信量及一条默认路由，允许的LSA为1,2,7

       末梢的配置：
       普通末梢的配置(所设计到的区域内路由器都需配置)：
	Router(config)#router ospf 1
	Router(config-router)#area  区域号  stub

       完全末梢的配置(所设计到的区域内路由器都需配置)：
	Router(config)#router ospf 1		// 在 ABR 上配置
	Router(config-router)#area  区域号  stub no-summary 

	Router(config)#router ospf 1		// 其他路由器配置
	Router(config-router)#area  区域号  stub

       非纯末梢（nssa）的配置：
	Router(config)#router ospf 1
	Router(config-router)#area  区域号  nssa

       完全非纯末梢配置（在ABR上配置）：
	Router(config)#router ospf 1
	Router(config-router)#area  区域号  nssa no-summary
              或
	Router(config)#router ospf 1
	Router(config-router)#area  区域号  nssa default-information originate

       查看OSPF链路状态数据库信息：
	Router#show ip ospf database

       OSPF 多区域配置的常用检查命令：
	show ip route			查看路由表信息（直连/学习）
	show ip route ospf			只查看 OSPF 学习到的路由
	show ip protocol			查看 OSPF 配置信息
	show ip ospf			查看在路由器上 OSPF 的配置方式以及 ABR 的信息
	show ip ospf database		查看 LSDB 内的所有 LSA 数据信息
	show ip ospf interface		查看接口上 OSPF 配置的信息（进程 ID、Route ID、Cost、Priority、邻居的数量）
	show ip ospf neighbor		查看 OSPF 邻居和邻接的状态（Full 表示邻居状态正常）
	show ip ospf neighbor detail		查看 OSPF 邻居的详细信息（包括 DR/BDR）
	debug ip ospf adj			查看路由器邻接的整个过程
	debug ip ospf packet		查看每个 OSPF 数据包的信息（包含 OSPF 版本、Route ID、Area ID）
	clear ip route			清空路由表

       补充：
	静态路由的管理距离默认是 1，度量值是 0
	RIP 协议的管理距离默认是 120，度量值是跳数
	OSPF 协议的管理距离默认是110，度量值是路径开销

6.    OSPF 高级配置（路由重分发）：

       将 OSPF 重分发给 RIP：
	Router(config)#router rip
	Router(config-router)#redistribute ospf 1 metric 跳数

       将RIP重分发给OSPF：
	Router(config)#router ospf 1
	Router(config-router)#redistribute rip subnets	// 不写cost，默认为20

       将静态重分发给OSPF：
	Router(config)#router ospf 1
	Router(config-router)#redistribute static subnets

       将默认重分发给OSPF：
	Router(config)#router ospf 1
	Router(config-router)#default-information originate

       将直连路由重分发给OSPF：
	Router(config)#router ospf 1
	Router(config-router)#redistribute connected subnets

8.    Cisco ASA（防火墙）：

       修改主机名
	hostname  名字
       特权密码
	enable  password   密码
       远程登录密码（telnet密码）
	passwd  密码
       清除所有配置
	全局模式下：clear   configure  all
       ACL配置
	access-list  名字  permit  ip  源网段  子网掩码（正码）   目标网段  子网掩码（正码）      
       规则应用：
	access-group  名字  in(out)  int  逻辑接口名
       接口配置(e0/0)
	int e0/0
	nameif inside
	security-level 100 
	ip add  192.168.1.1 255.255.255.0
	no sh
		// inside 默认安全级别为100，outside 默认安全级别为0
       静态路由和默认路由配置
	静态： route  inside (或outside)  目标网段  子网掩码  下一跳
	默认：route  inside (或outside)  0.0.0.0  0.0.0.0   下一跳
       telnet配置
	telnet  网段  子网掩码  inside(或outside)
	telnet  0  0 inside  (允许所有的主机)
       ssh配置
	host asa802  （配置主机名）
	domain-name asadomain.com  （配置域名）
	crypto key generate rsa modulus 1024  （使用RSA加密）
	ssh 0 0 outside  （允许所有主机从outside访问）
	ssh timeout 30 （超时等待时间）
	ssh version 2  （声明版本）

       ASDM配置
	http server enable （启用功能）
	http 0 0 outside （允许所有主机从outside访问）
	asdm image disk0:/asdm-602.bin  （提供给客户端的镜像文件）
	username benet password cisco privilege 15  （创建用户名和密码，声明级别，15是管理员的身份）

       查看conn表
	show conn  detail

	show  route

9.    Cisco ASA 应用 NAT：

       动态NAT配置（内访问外，多对多）：
       定义内部inside网段： nat (inside) 1  网段  子网掩码   
                       或：  nat  (inside)  1  0   0  (代表内部所有网段)
       定义外部outside映射地址：global (outside)  1  起始IP地址-结束IP地址
              如：global  (outside)  1   192.168.1.100-192.168.1.200

       动态PAT配置（内访问外，多对一）：
       方法一（映射到外部的一个IP地址）：
	定义内部inside网段： nat (inside) 1  网段  子网掩码   
                       nat  (inside)  1  0   0  (代表内部所有网段) 
  	定义外部outside映射地址：global (outside)  1  ip地址
                       如：global (outside)  1  192.168.1.100
       方法二（映射到外部的接口上）：
	定义内部inside网段： nat (inside) 1  网段  子网掩码   
                       	     nat  (inside)  1  0   0  (代表内部所有网段) 
	定义外部outside映射接口：global (outside)  1  interface

3、静态NAT配置（发布服务器，一对一）：
	static (dmz,outside) 映射的IP地址  内部服务器IP地址       
	如：static (dmz,outside)  200.0.0.10 192.168.1.10
    还需配置ACL（outside能访问dmz）:
      access-list out_to_dmz permit ip   any(外部主机地址)   host  200.0.0.10(映射的IP地址) 
      access-group out_to_dmz in int outside

	static (dmz,outside)  tcp  映射的IP地址  端口号  内部服务器IP地址  端口号      
	如：static (dmz,outside) tcp  200.0.0.10  80  192.168.1.10   80 
    还需配置ACL（outside能访问dmz）:
      access-list out_to_dmz permit ip   any(外部主机地址)   host  200.0.0.10(映射的IP地址) 
      access-group out_to_dmz in int outside
