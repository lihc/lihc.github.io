基于二进制日志文件位置的复制

一、主从复制的概述

官方文档：https://dev.mysql.com/doc/refman/5.7/en/binlog-replication-configuration-overview.html

MySQl 主从复制的原理：

基于二进制日志文件位置的复制，作为主服务器运行的 MySQL 实例将更新和更改作为 “事件” 写入二进制日志。根据记录的数据库杠，二进制日志中的信息以不同的日志格式存储。从服务器读取主服务器二进制日志并在本地数据库上执行二进制日志中的事件。

二进制日志中的事件使用多种格式记录：

  • 基于语句的复制
  • 基于行的复制
  • 混合格式的复制

每个从服务器都需要记录二进制日志的坐标，每个服务器必须配置一个唯一的 ID。

二、主从复制的搭建

1. 主服务器配置

关闭防火墙：

systemctl stop firewalld.service 
systemctl disable firewalld.service

修改 my.cnf 文件：

# /etc/my.cnf
[mysqld]
log-bin=mysql-bin
# 启用二进制日志
server-id=1
# 设置服务器ID=1,当 ID=0 时,主服务器拒绝连接到从服务器

重启 MySQL 服务：

systemctl restart mysqld

创建复制用户：

# 创建用户并授权
mysql> CREATE USER 'repl'@'%.example.com' IDENTIFIED BY 'password';
mysql> GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%.example.com';
#mysql> GRANT REPLRCATION SLAVE ON *.* TO 'root'@'192.168.10.20' IDENTIFIED BY '1234.com';

# 刷新权限
mysql> FLUSH PRIVILEGES;

获取二进制日志坐标：

# 刷新所有表并阻止写入操作
mysql> FLUSH TABLES WITH READ LOCK;

# 查看主服务器状态
mysql> SHOW MASTER STATUS;
+------------------+----------+--------------+------------------+-------------------+
| File             | Position | Binlog_Do_DB | Binlog_Ignore_DB | Executed_Gtid_Set |
+------------------+----------+--------------+------------------+-------------------+
| mysql-bin.000001 |      771 |              |                  |                   |
+------------------+----------+--------------+------------------+-------------------+
1 row in set (0.00 sec)
# File为日志文件名称，Postion为日志文件坐标

# 释放锁
mysql> UNLOCK TABLES

2. 从服务器配置

关闭防火墙：

systemctl stop firewalld.service 
systemctl disable firewalld.service

修改 my.cnf 文件：

# /etc/my.cnf
[mysqld]
server-id=2

重启 MySQL 服务：

systemctl restart mysqld

配置从服务器：

mysql> CHANGE MASTER TO
    -> MASTER_HOST='source_host_name',
    -> MASTER_USER='replication_user_name',
    -> MASTER_PASSWORD='replication_password',
    -> MASTER_LOG_FILE='recorded_log_file_name',
    -> MASTER_LOG_POS=recorded_log_position;

启动 Slave 线程：

start slave;

检查从服务器状态：

show slave status \G

Slave_IO_Running: Yes

Slave_SQL_Running: Yes
