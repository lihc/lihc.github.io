CentOS 安装 MySQL

一、使用二进制文件安装 MySQL

CentOS 7.6 使用二进制包安装 MySQL 5.7.32。

官方文档：https://dev.mysql.com/doc/refman/5.7/en/binary-installation.html

下载安装文件

软件地址：https://downloads.mysql.com/archives/community/

  • Product Version： 5.7.32
  • Operating System：Linux - Generic

选择合适的版本，本篇使用软件包 mysql-5.7.32-linux-glibc2.12-x86_64.tar.gz 为例进行安装。

解决依赖关系

删除配置文件：

rm -rf /my.cnf*

MySQL 依赖于该libaio 库：

# Yum安装方法
yum install libaio
# APT安装方法
apt-get install libaio1

创建用户和组

groupadd mysql
useradd -r -g mysql -s /bin/false mysql

解压缩软件包

tar zxf mysql-5.7.32-linux-glibc2.12-x86_64.tar.gz -C /usr/local/
mv /usr/local/mysql-5.7.32-linux-glibc2.12-x86_64 /usr/local/mysql

cd /usr/local/mysql/
mkdir mysql-files
chown mysql:mysql mysql-files
chmod 750 mysql-files

配置环境变量

echo 'export PATH=$PATH:/usr/local/mysql/bin' >> /etc/profile
export PATH=$PATH:/usr/local/mysql/bin

初始化数据库

bin/mysqld --initialize --user=mysql
#初始化取消密码，使用"--initialize-insecure"参数
bin/mysql_ssl_rsa_setup
bin/mysqld_safe --user=mysql &

添加系统服务

cp /usr/local/mysql/support-files/mysql.server /etc/init.d/mysql.server

二、使用源代码编译安装 MySQL

CentOS 7.6 使用源代码编译安装 MySQL 5.7.32。

官方文档：https://dev.mysql.com/doc/refman/5.7/en/source-installation.html

下载安装文件

软件地址：https://downloads.mysql.com/archives/community/

Product Version：5.7.32

Operating System：Source Code

OS Version：Generic Linux (Architecture Independent)

选择 mysql-5.7.32.tar.gz 软件包进行下载。

解决依赖关系

源码安装先决条件：

  • CMake，在所有平台上用作构建框架。CMake可以从http://www.cmake.org下载。强烈建议使用 GNU make 3.75 或更高版本。

  • GCC：一个有效的 ANSI C++ 编译器。gcc 是Linux下的C语言编译工具。

  • OpenSSL：需要 SSL 库来支持加密连接，用于生成随机数的熵以及其他与加密有关的操作。

  • Boost C++：需要 Boost C++ 库来构建 MySQL（但不使用它）。必须安装 Boost 1.59.0。要获取 Boost 及其安装说明，请访问 官方网站。安装 Boost 后，通过WITH_BOOST在调用 CMake 时定义选项来告诉构建系统 Boost 文件所在的位置 。例如：

    cmake . -DWITH_BOOST=/usr/local/boost_1_59_0
#根据需要调整路径以匹配您的安装。

  • ncurses：字符终端处理库。依赖 ncurses。

要从标准源发行版安装 MySQL，需要解压工具：

yum -y install gunzip

要从开发源树安装 MySQL，需要以下附加工具：

  • Git：分布式版本控制系统

  • Bison：Linux 下 C/C++ 语法分析器。

  • 在Solaris Express上，除了 bison 之外，还必须安装 m4。

安装依赖软件：

yum -y install cmake gcc gcc-c++ openssl-devel ncurses ncurses-devel bison

安装 Boost C++：

wget https://sourceforge.net/projects/boost/files/boost/1.59.0/boost_1_59_0.tar.gz/download -O boost_1_59_0.tar.gz
tar zxf boost_1_59_0.tar.gz -C /usr/local/

创建用户和组

groupadd mysql
useradd -r -g mysql -s /bin/false mysql

编译安装 MySQL

解压缩软件包：

tar zxf mysql-5.7.32.tar.gz -C /usr/local/src/
cd /usr/local/src/mysql-5.7.32/

创建工作目录：

#为了保持源代码树清洁，创建 build 目录
mkdir build
cd build/

编译 MySQL：

cmake .. -DCMAKE_INSTALL_PREFIX=/usr/local/mysql \
-DMYSQL_DATADIR=/usr/local/mysql/data \
-DSYSCONFDIR=/etc \
-DDEFAULT_CHARSET=utf8 \
-DDEFAULT_COLLATION=utf8_general_ci \
-DEXTRA_CHARSETS=all \
-DMYSQL_UNIX_ADDR=/tmp/mysql.sock \
-DWITH_MYISAM_STORAGE_ENGINE=1 \
-DWITH_INNOBASE_STORAGE_ENGINE=1 \
-DWITH_ARCHIVE_STORAGE_ENGINE=1 \
-DWITH_PARTITION_STORAGE_ENGINE=1 \
-DWITH_SYSTEMD=1 \
-DWITH_BOOST=/usr/local/boost_1_59_0

MySQL配置选项描述：

选项描述
-DCMAKE_INSTALL_PREFIX=dir_name基础安装目录。
-DMYSQL_DATADIR=dir_nameMySQL数据目录的位置。
-DSYSCONFDIR=dir_name默认my.cnf选项文件目录。
-DDEFAULT_CHARSET=charset_name服务器字符集，默认latin1（cp1252西欧）字符集。
-DDEFAULT_COLLATION=collation_name服务器排序规则。默认使用 latin1_swedish_ci。
-DEXTRA_CHARSETS=all使MySQL支持所有的扩展字符。
-DMYSQL_UNIX_ADDR=file_nameUnix 套接字文件路径。必须是绝对路径名。默认值为/tmp/mysql.sock。
-DWITH_MYISAM_STORAGE_ENGINE=1添加 MYISAM 引擎支持。
-DWITH_INNOBASE_STORAGE_ENGINE=1添加 InnoDB 引擎支持。
-DWITH_ARCHIVE_STORAGE_ENGINE=1添加 ARCHIVE 引擎支持。
-DWITH_PARTITION_STORAGE_ENGINE=1安装支持数据库分区。
-DWITH_SYSTEMD=bool是否启用安装systemd支持文件，默认情况下，此选项是禁用的。
-DWITH_BOOST=path_nameBoost 库是构建MySQL所必需的。

更多选项：https://dev.mysql.com/doc/refman/5.7/en/source-configuration-options.html

安装 MySQL：

make && make install

初始化数据库

设置文件权限

chown -R mysql:mysql /usr/local/mysql

优化执行路径

echo 'export PATH=$PATH:/usr/local/mysql/bin' >> /etc/profile
source /etc/profile

修改配置文件

cat <<EOF > /etc/my.cnf
[mysqld]
basedir=/usr/local/mysql
datadir=/usr/local/mysql/data
socket=/tmp/mysql.sock
log-error=/usr/local/mysql/data/mysql.err
pid-file=/usr/local/mysql/data/mysqld.pid
EOF

初始化数据库

mysqld --initialize --user=mysql
#初始化取消密码，添加"--initialize-insecure"参数

查看数据库密码

cat /usr/local/mysql/data/mysql.err | grep password

输出结果类似：

2020-11-20T04:25:45.535680Z 1 [Note] A temporary password is generated for root@localhost: W2l0DdwcpI/W

修改数据库密码：

ALTER USER root@localhost identified by '1234.com';

添加系统服务

创建服务文件：

cat <<EOF > /usr/lib/systemd/system/mysqld.service
[Unit]
Description=MySQL Server
Documentation=man:mysqld(8)
Documentation=http://dev.mysql.com/doc/refman/en/using-systemd.html
After=network.target
After=syslog.target

[Install]
WantedBy=multi-user.target

[Service]
User=mysql
Group=mysql
Type=forking
PIDFile=/usr/local/mysql/data/mysqld.pid
TimeoutSec=0
PermissionsStartOnly=true
ExecStartPre=/usr/local/mysql/bin/mysqld_pre_systemd
ExecStart=/usr/local/mysql/bin/mysqld --daemonize --pid-file=/var/run/mysqld/mysqld.pid $MYSQLD_OPTS
EnvironmentFile=-/etc/sysconfig/mysql
LimitNOFILE = 5000
Restart=on-failure
RestartPreventExitStatus=1
PrivateTmp=false
EOF
#详细解释查看"/usr/local/src/mysql-5.7.32/build/scripts/mysqld.service"文件

systemctl daemon-reload
