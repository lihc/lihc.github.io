1. Web配置邮件报警

  1. Web启用Report Problems to Zabbix Administrators

    配置——动作——启用Report——应用

  2. 在Web界面配置添加收件人用于接收邮件

    用户基本资料: Zabbix Administrator（右上角👨‍💼）——报警媒介

    类型收件人当启用时如果存在严重性则使用Status动作
    Email邮箱地址1-7,00:00-24:00未 | 信 | 警 | 一 | 严 | 灾已启用

  3. 配置发件人

    管理——报警媒介类型——Email

    名称类型SMTP服务器SMTP服务端口HELOSMTP电邮安全链接用户名称密码
    Email电子邮件SMTP.163.com465163.com邮箱地址SSL用户名授权码
    • SMTP服务器：QQ邮箱（smtp.qq.com） 网易163邮箱（smtp.163.com）
    • SMTP服务端口：SMTP (465)
    • HELO：QQ邮箱(qq.com)，网易邮箱(163.com)。向服务器传递的第一个命令是HELO。该命令包含一个参数，即你的邮箱名
    • 密码：目前邮件服务商大都采用授权码，很少量采用密码登录

  4. 微信关联QQ邮箱，可以实现微信报警

2. 配置邮件脚本报警

？？：Web界面已经可以使用邮件报警了，为什么还要使用脚本？
！！：部分企业服务器版本以及Zabbix版本太低，不支持Web邮件报警！

  1. Yum安装邮件服务

    关闭Zabbix-Server上的邮件服务，安装Mailx

    systemctl stop postfix && systemctl disable postfix
    yum -y install mailx

  2. 编辑 Mailx 的配置

    vim /etc/mail.rc
···
set from=邮箱地址
set smtp=smtps://smtp.163.com:465
set smtp-auth-user=邮箱用户
set smtp-auth-password=授权码
set smtp-auth=login
set ssl-verify=ignore
set nss-config-dir=/etc/pki/nssdb/

    测试 Mailx 配置

    echo This is a Test. | mail -s Test 接收方邮件地址

  3. 编写邮件报警脚本

    官方文档：https://www.zabbix.com/documentation/4.0/zh/manual/config/notifications/media/script

    查看配置文件定义的脚本路径

    cat /etc/zabbix/zabbix_server.conf | grep ^AlertScriptsPath
AlertScriptsPath=/usr/lib/zabbix/alertscripts
    vim /usr/lib/zabbix/alertscripts/Email_Script.sh
#!/bin/bash

to=$1
subject=$2
body=$3

cat <<EOF | mail -s "$subject" "$to"
$body
EOF

    赋予脚本执行权限

    chmod +x /usr/lib/zabbix/alertscripts/Email_Script.sh

    测试邮件脚本

    bash /usr/lib/zabbix/alertscripts/Email_Script.sh 收件人邮箱地址 Test "This is a Test"

  4. Zabbix Web控制台关联邮件报警脚本

    管理——报警媒介类型(关闭已有的Email)——创建媒体类型——创建

    名称类型脚本名称脚本参数
    邮件脚本报警脚本Email_Script.sh“{ALERT.SENDTO}”、”{ALERT.SUBJECT}”、”{ALERT.MESSAGE}”

  5. 创建报警媒介(指定脚本报警收件人)

    用户基本资料: Zabbix Administrator（右上角👨‍💼）——报警媒介

    类型收件人当启用时如果存在严重性则使用Status动作
    邮件脚本报警邮箱地址1-7,00:00-24:00未 | 信 | 警 | 一 | 严 | 灾已启用

  6. 对邮件脚本进行优化

    做完以上步骤，已经可以使用脚本报警了，但是报警信息将会使用附件的方式进行发送，不利于查看，所以我们对邮件脚本进行优化

    yum -y install dos2unix
vim /usr/lib/zabbix/alertscripts/Email_Script.sh
    #!/bin/bash
#export UTF-8
FILE=/tmp/mail.txt
echo "$3" > $FILE
dos2unix -k $FILE
/bin/mail -s "$2" $1 < $FILE

    修改脚本完成保存退出即可！

    PS：这个脚本可以进行本地测试，但测试完成后一定要将/tmp/mail.txt文件删除

3. 设置企业微信报警

微信报警
1.申请企业微信
	我的企业——复制企业ID
	应用管理——创建应用——名称：报警机器人——选择部门——创建应用——报警机器人——复制：AgentID，Secret。

2.编写Python微信报警脚本，并修改企业id和用户密码
	vim /usr/lib/zabbix/alertscripts/WeiXin.py
	修改：
	corpid='ID号'		#企业ID
	appsecret='机器人密码'   #报警机器人密码
	agentid=机器人ID					#报警机器人ID
	保存退出
    赋予执行权：chmod 777 weixin.py
	格式转换：dos2unix -k winxin.py
	赋权weixin.log: chown zabbix:zabbix /tmp/weixin.log

	测试发送微信：./weixin.py 微信号  Test "This is a Test."

3.Zabbix Web关联
管理——报警媒介类型——右上角“创建媒体类型”——“名称（微信报警），类型（脚本），脚本名称（必须写对），参数添加3个：{ALERT.SENDTO}、{ALERT.SUBJECT}、{ALERT.MESSAGE}”——更新

右上角——小人（用户）——报警媒介——“类型（微信报警，收件人：微信账号）”——添加——更新
#!/usr/bin/env python
#-*- coding: utf-8 -*-
import requests
import sys
import os
import json
import logging

logging.basicConfig(level = logging.DEBUG, format = '%(asctime)s, %(filename)s, %(levelname)s, %(message)s',
datefmt = '%a, %d %b %Y %H:%M:%S',
filename = os.path.join('/tmp','weixin.log'),
filemode = 'a')

corpid='微信企业号corpid'
appsecret='应用的Secret'
agentid=应用的id
#获取accesstoken
token_url='https://qyapi.weixin.qq.com/cgi-bin/gettoken?corpid=' + corpid + '&corpsecret=' + appsecret
req=requests.get(token_url)
accesstoken=req.json()['access_token']

#发送消息
msgsend_url='https://qyapi.weixin.qq.com/cgi-bin/message/send?access_token=' + accesstoken

touser=sys.argv[1]
subject=sys.argv[2]
#toparty='3|4|5|6'
message=sys.argv[2] + "\n\n" +sys.argv[3]

params={
"touser": touser,
# "toparty": toparty,
"msgtype": "text",
"agentid": agentid,
"text": {
"content": message
},
"safe":0
}

req=requests.post(msgsend_url, data=json.dumps(params))
logging.info('sendto:' + touser + ';;subject:' + subject + ';;message:' + message)

自定义模版监控Nginx状态

所需环境：Nginx服务器(192.168.10.40)，并安装Zabbix-Agent，修改配置文件指向Zabbix监控服务器

安装Nginx服务

yum -y install epel-release
yum -y install nginx

  1. 开启监控页面并访问测试

    cat /etc/nginx/nginx.conf 
定位到49行，添加：
        location /nginx_status {
           stub_status on;
           access_log off;
        }
重启服务：systemctl restart nginx
    curl 127.0.0.1/nginx_status/
Active connections: 1 
server accepts handled requests
 1 1 1 
Reading: 0 Writing: 1 Waiting: 0

  2. 准备Nginx监控状态脚本

    cat /etc/zabbix/zabbix_agentd.d/nginx_monitor.sh
#!/bin/bash
NGINX_COMMAND=$1
CACHEFILE="/tmp/nginx_status.txt"
CMD="/usr/bin/curl http://127.0.0.1/nginx_status/"
if [ ! -f $CACHEFILE  ];then
   $CMD >$CACHEFILE 2>/dev/null
fi
# Check and run the script
TIMEFLM=`stat -c %Y $CACHEFILE`
TIMENOW=`date +%s`
if [ `expr $TIMENOW - $TIMEFLM` -gt 60 ]; then
    rm -f $CACHEFILE
fi
if [ ! -f $CACHEFILE  ];then
   $CMD >$CACHEFILE 2>/dev/null
fi
nginx_active(){
         grep 'Active' $CACHEFILE| awk '{print $NF}'
         exit 0;
}
nginx_reading(){
         grep 'Reading' $CACHEFILE| awk '{print $2}'
         exit 0;
}
nginx_writing(){
         grep 'Writing' $CACHEFILE | awk '{print $4}'
         exit 0;
}
nginx_waiting(){
         grep 'Waiting' $CACHEFILE| awk '{print $6}'
         exit 0;
}
nginx_accepts(){
         awk NR==3 $CACHEFILE| awk '{print $1}' 
         exit 0;
}
nginx_handled(){
         awk NR==3 $CACHEFILE| awk '{print $2}' 
         exit 0;
}
nginx_requests(){
         awk NR==3 $CACHEFILE| awk '{print $3}'
         exit 0;
}
case $NGINX_COMMAND in
    active)
        nginx_active;
        ;;
    reading)
        nginx_reading;
        ;;
    writing)
        nginx_writing;
        ;;
    waiting)
        nginx_waiting;
        ;;
    accepts)
        nginx_accepts;
        ;;
    handled)
        nginx_handled;
        ;;
    requests)
        nginx_requests;
        ;;
    *)
echo 'Invalid credentials';
exit 2;
esac

    赋予执行权

    chmod +x /etc/zabbix/zabbix_agentd.d/nginx_monitor.sh

  3. 编写Zabbix监控配置文件

    cat /etc/zabbix/zabbix_agentd.d/nginx_status.conf
UserParameter=nginx_status[*],/bin/bash /etc/zabbix/zabbix_agentd.d/nginx_monitor.sh $1

    重启Agent客户端

    systemctl restart zabbix-agent.service

    修改缓存文件权限

    chown zabbix:zabbix /tmp/nginx_status.txt

  4. Zabbix服务器使用Zabbix_Get取值

    zabbix_get -s 192.168.10.40 -k nginx_status[accepts]

  5. 导入模板——链接模板——查看数据

    配置——主机——Nginx——选择“全克隆”——填写新Nginx的IP
